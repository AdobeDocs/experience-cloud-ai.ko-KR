# Playground에서 Coworker 채팅 시작

플레이그라운드에서 동료 채팅을 사용하여 자연어 프롬프트가 작업을 학습, 조사 및 구체화하는 데 어떻게 도움이 될 수 있는지 살펴보십시오. Playground는 동료 채팅이 수행할 수 있는 작업을 빠르게 이해할 수 있도록 대화를 시작하는 샘플 방법을 제공합니다.

>[!IMPORTANT]
>
>다음을 사용하여 이 연습을 완료합니다.
>
>- 링크: [ao.adobe.io](https://ao.adobe.io/)
>- 인스턴스: AEP GenAI - VA7
>- 샌드박스: `fsi-box`

## AI 동료 데모: [!DNL weBank] 신용 카드 업그레이드 캠페인

이 연습에서는 Real-Time CDP 및 Adobe Journey Optimizer을 다룹니다.

**[!DNL weBank]**&#x200B;의 CX 엔터프라이즈 전략가인 디지털 기반 금융 서비스 회사입니다. AI 동료를 사용하여 원시 데이터에서 라이브 캠페인으로 전환하고, 완전히 대화를 통해 전환할 수 있습니다.

## 3부: 재활성화 캠페인에 적합한 대상 찾기

### 3.1 — 적합한 대상을 찾을 수 있도록 AI에 요청

**프롬프트**

> 신용 카드 재활성화 캠페인에 적합한 대상을 찾거나 만들 수 있도록 지원 — 신용 카드 업그레이드에 대해 높은 성향 또는 중간 성향으로 점수를 받았지만 아직 조치를 취하지 않은 고객을 타깃팅하고 싶습니다

**예상 결과**

동료는 여정 또는 대상에서 타겟팅되지 않은 두 개의 후보 대상을 찾습니다.

| 대상자 | 프로필 |
| --- | --- |
| 신용 카드 업그레이드 높음 | 1,195 |
| 신용 카드 업그레이드 Medium | 4,113 |

권장 옵션:

| 옵션 | 접근 방식 |
| --- | --- |
| **A** | 여정 분기(높음, Medium에 대한 적극적인 오퍼)와 함께 두 가지를 모두 직접 사용합니다. |
| **B** | 효율적인 관리를 위해 업그레이드 제외 논리를 사용하여 파생된 세그먼트 만들기 |

결합된 크기: 약 5,308개의 프로필

### 3.2 — 정밀도 대상 만들기

**프롬프트**

> &quot;성향: 신용 카드 업그레이드 높음&quot; 대상에 속하고 계정 값이 25만 달러 이상이고 이메일을 선택한 고객에 대한 대상을 만듭니다

이름을 **플래티넘 업그레이드 - 고가치 전자 메일 적격**(으)→ 일괄 처리 → 승인 계획 선택

**예상 결과**

동료는 ML 성향, 재무 가치 및 채널 자격을 하나의 정밀도 세그먼트로 계층화합니다. 이 세그먼트에서는 일반적으로 데이터 팀 티켓이 필요한 작업입니다.

### 3.3 — 예상 크기 및 폭포

**프롬프트**

> 대상이 얼마나 됩니까? 폭포 분류 좀 보여 주세요.

**예상 결과**

Coworker는 실시간 프로필 수 예상 값과 시각적 폭포 차트를 반환합니다.

| 필터 단계 | 결과 |
| --- | --- |
| 신용 카드 업그레이드 높은 성향 | 약 1,195개의 프로필 |
| + 25만 달러 이상의 계정 가치 | 더 좁혀짐 |
| + 이메일 옵트인 | 최종 주소 지정 가능 수 |

이는 어떤 조건이 가장 큰 필터인지, 기준을 느슨하게 할지 여부를 정확하게 보여 줍니다.

### 3.4 — 유사한 대상 확인

**프롬프트**

> 방금 만든 대상과 유사한 기존 대상이 있습니까?

**예상 결과**

동료는 결합된 단일 대상이 존재하지 않지만 모든 구성 요소가 이미 존재함을 확인합니다.

| 빌딩 블록 | 세부 사항 |
| --- | --- |
| 성향: 신용 카드 업그레이드 높음 | 점수 ≥ 90 |
| 계정 값: 250-500K | 기존 대상자 |
| 계정 값: +500K | 기존 대상자 |
| 이메일 옵트인 | `consents.marketing.email.val = "y"` |

동료는 중복 위험이 없음을 확인하고 계속 진행할 것을 권장합니다.

## 4부: 여정 구축

### 4.1 — 데이터에 기반을 둔 여정 아이디어 얻기

**프롬프트**

> 금융 서비스의 인기 있는 사용 사례를 기반으로, 보유하고 있는 대상자와 함께 만들 수 있는 빠른 성공 여정을 제안하십시오

**예상 결과**

동료는 실제 대상자에게 기반을 둔 다섯 가지 구체적인 여정 아이디어를 제안합니다.

| 여정 아이디어 | 대상자 |
| --- | --- |
| 생일 충성도 터치 | 7일 이내의 생일 + 계정 값 계층 |
| 신용 카드 업그레이드 지원 | CC 업그레이드 높음 + 이메일 옵트인 |
| 모기지 지역 지원 | 모기지 상한 + 동부/서부 해안 |
| 자산 계층 자문 업그레이드 | 계정 값: +500K |
| 옵트인 크로스 채널 활성화 | 이메일 옵트인 - SMS 옵트인 |

### 4.2 — 신용 카드 업그레이드 여정 만들기

**프롬프트**

> &quot;플래티넘 업그레이드 - 고가치 이메일 적격&quot; 대상을 사용하여 여정을 만듭니다. 위뱅크 플래티넘 카드 혜택을 소개하는 푸시 알림을 보냅니다. 3일 정도 기다리십시오. 고객이 신청하지 않은 경우 잔고 이체 시 12개월 동안 0% APR의 제한 시간 오퍼로 SMS를 보내십시오. 5일 더 기다리십시오. 아직 신청이 없는 경우 개인 은행원 초대를 받아 최종 푸시 알림을 보내십시오.

계획 검토 → 계획 승인 → 권한 부여

**예상 결과**

동료는 대기 타이머, 조건 확인 및 에스컬레이트된 여정을 포함한 단일 자연어 설명으로 전체 3터치 2채널 오퍼를 만듭니다.

### 4.3 — 보너스: 이미지에서 여정 만들기

**프롬프트**

> 내가 업로드한 이미지에서 이 여정 만들기

**예상 결과**

동료는 업로드된 이미지(노드, 채널, 대기 시간 및 조건 포함)를 읽고 Journey Optimizer에서 전체 여정을 생성합니다. 이렇게 하면 계획 아티팩트가 배포 가능한 여정으로 직접 전환됩니다.

### 4.4 — 충돌 확인

**프롬프트**

> 방금 만든 여정과 충돌할 수 있는 활성 여정이 있습니까?

**예상 결과**

Coworker는 모든 활성 여정에서 대상 겹침, 충돌 예약 및 채널 채도를 자동으로 확인합니다.

## 5부: 여정 드롭오프를 새 대상으로 전환

### 5.1 — 드롭오프 지점 식별

**프롬프트**

> 방금 만든 여정을 보여 주고 드롭오프가 가장 큰 단계를 식별합니다

**예상 결과**

동료는 여정 단계 이벤트를 분석하고 종료 비율이 가장 높은 노드를 표시합니다. 예:

> 첫 번째 이메일을 받은 프로필의 68%가 이메일을 열지 않고 SMS 단계 전에 종료했습니다.

### 5.2 — 드롭오프 대상 만들기

**프롬프트**

> 이러한 여정 드롭오프를 새로운 대상자(플래티넘 업그레이드 여정에 참여했지만 SMS 단계에 도달하지 않은 사용자)로 전환합니다

이름을 **플래티넘 업그레이드 - 비응답자에게 전자 메일 보내기** → 배치 → 승인 계획 선택

**예상 결과**

동료는 여정 동작 데이터(여정에 들어갔지만 SMS 단계로 진행되지 않은 프로필)를 기반으로 새로운 대상을 만듭니다. 이 대상은 즉시 후속 캠페인에 사용할 수 있습니다.

### 5.3 — 드롭오프 재활성화

**프롬프트**

> &quot;플래티넘 업그레이드 - 이메일 비응답자&quot; 대상에 대한 재활성화 여정을 만듭니다. 다른 방법을 사용해 보십시오. 제한된 시간 동안 제공되는 75,000개의 보너스 포인트를 강조 표시하는 푸시 알림으로 시작하여 2일을 기다린 다음 DM 초대를 보내어 현지 지점을 방문하십시오.

계획 검토 → 계획 승인 → 권한 부여

**예상 결과**

동료는 비응답자를 위해 다양한 채널과 메시징을 사용하여 전용 복구 여정을 만듭니다. 이제 완전히 대화를 통해 전환 누출에 대한 루프를 닫았습니다.

## 6부: 자동화된 QA 및 유효성 검사

### 6.1 — 데이터 새로 고침 및 수집 상태 확인

**프롬프트**

> 대상자에게 데이터를 제공하는 데이터 세트에 데이터 품질 문제가 있습니까? weBank: CRM 및 weBank: 고객 액션에 대한 수집 상태 확인

**예상 결과**

동료가 각 데이터 세트에 대한 수집 상태를 보고합니다.

| 데이터 세트 | 상태 | 세부 사항 |
